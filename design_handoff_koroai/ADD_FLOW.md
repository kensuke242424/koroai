> 食品追加モーダルを「**選ぶ**」と「**確認・編集**」の2画面に分離した刷新版の仕様。
> README.md の「② 食べものを追加」「③ 追加 — 詳細入力」を**本書で置き換え**ます。

> - 実装ファイル: `prototype/fk-flows.jsx` の `FKAddSheet` / `FKConfirmItem`

## 全体像
ホームの FAB（＋）で**背の高いボトムシート**が開く（`FKSheet height="88%"`）。シート内は横プッシュ遷移の2画面：

```
[選ぶ]  --「確認する（N品）」-->  [確認・編集]  --「冷蔵庫に追加（N品）」--> ホーム
   ^------------------ 「‹ 選び直す」 --------------------|
```

シート内部は `width:200%` のトラックを `translateX(0 / -50%)` で切り替える（`.32s`）。

## 画面1：選ぶ
- **目的**: 買ってきた食材を素早くカゴに入れる（速さ優先・詳細は入れない）。
- **構成**:
  - ヘッダー: 「何を買ってきた？」＋サブ「カテゴリを選んでカゴへ。最後にまとめて確認します。」
  - **カテゴリタイル（種類別セクション）**: 肉・魚介／野菜・きのこ／果物・乳製品／大豆製品・卵／主食・惣菜（どれにも入らないものは「その他」へ自動）。3列グリッド、円形グリフ、右上にカウントバッジ（未選択「＋」／選択時カテゴリ色で個数）。
  - **タイルをタップ＝カゴに1つ追加**（詳細入力には飛ばない）。同カテゴリを複数回タップでカウント加算。
  - **下部カゴエリア（別色トレイ）**: `var(--fk-surface2)` 地＋角丸トップ＋上向きの淡い影で独立した領域。中身：
    - 左に🧺（固定）、右に**カウントチップを横スクロール**。
    - チップ＝カテゴリ集約。左にカウントバッジ、ラベル、✕。**最新の選択が左**から積まれる（追加時ポップイン `fkChipIn`）。
    - **✕**: カウント2以上なら−1、1なら削除。**タイルのカウントとチップのカウントは常に一致**。
    - スクロールバーは細い3px（`.fkChipTray`）。
    - 「**確認する（N品）**」ボタン（カゴが空のときは不活性）。

## 画面2：確認・編集
- **目的**: カゴの各食材を落ち着いて確認・編集してから登録。
- **構成**:
  - ヘッダー: 「‹ 選び直す」／「内容を確認」、サブ「名前・もち日数・残量を確認して追加できます。」
  - **カテゴリ別セクション**で各食材を**カード**表示（`FKConfirmItem`）。1カード＝1アイテム。
    - **名前**: テキスト入力（空ならカテゴリ既定名 `FK_CAT_DEFAULT_NAME` で補完）。
    - **もち日数**: ステッパー（−／あとN日／＋）＋**📅トグル**。📅を押すと**インライン・ミニカレンダー**が高さアニメ（`grid-template-rows 0fr⇄1fr`, `.3s`）で開閉。日付タップで日数に変換（過去日は不可、今日にリング、ステッパーと双方向同期）。
    - **残量**: `FKAmtSection`。モードトグル「ざっくり量／個数」を常設。ざっくり量＝吸着スライダー（わずか/少し/半分/多め/満タン）、個数＝シンプルカウント（単位はカテゴリ別）。
    - **✕**でそのカードを除外（最後の1つを消すと選ぶ画面へ戻る）。
  - 末尾に「＋ 食べものを選び直す」（破線）→ 選ぶ画面へ。
  - 下部に同じ別色トレイ＋「**冷蔵庫に追加（N品）**」。

## データ・コミット
- カゴ要素は `window.FKD.fkMakeItem(catId, name, days, { amtMode, amt, qty, qtyTotal, unit })`。
- タイル追加時の既定: `amtMode = FK_CAT_AMT_MODE[id] || 'amount'`、`amt: 0.72`、`qty: 1`、`unit = FK_CAT_UNIT[id] || '個'`、`days = カテゴリ既定`。
- 登録（commit）: `cart.forEach(it => onAdd({ ...it, name: 名前 || 既定名, qtyTotal: max(qtyTotal, qty) }))` → `onBatchDone(cart.length)` → `onClose()`。
- 非空のカゴを閉じようとすると確認ダイアログ（登録 or 破棄）。

## 再利用コンポーネント（既存）
`FKSheet`・`FKCatTile`・`FKStepper`・`FKDatePicker`・`FKCalToggle`・`FKAmtSection`（＋ `FKAmtModeToggle`/`FKAmtSlider`/`FKAmtCount`）・`FKIcon`。新規は `FKConfirmItem` と `FKAddSheet` の作り直しのみ。

## 実装メモ
- 本体は `FKSheet` が固定高さ（88%）のため**大detent相当の固定**。参照HTMLにある「中⇄大のドラッグdetent」は任意。本番環境のシート（iOS の detent 等）で表現するのが自然。
- 旧称 `FreshKeep` が残る箇所は「ころあい（Koroai）」に統一。
- 期限色は赤を使わず urgency ロジックを踏襲（README 参照）。残量は未設定可・モード切替を維持。
