# Handoff: ころあい（Koroai）— 消費期限管理・食品ロス削減アプリ

## Overview
「ころあい」は、冷蔵庫の食材を **食べ頃のうちに使いきる** ことを応援するモバイルアプリ（iOS 想定）です。アプリ名は「頃合い＝ちょうどいい食べ頃」に由来します。責めずにそっと寄り添うトーンで、食品ロスを罪悪感ではなくやさしさで減らすことをコンセプトにしています。

中心となる体験ループは **「足す → 食べ頃を知る → 食べきりを記録する」**。ホーム（今日の冷蔵庫）を起点に、食材の追加・編集・残量調整・食べきり記録・ふりかえりが回ります。

> ⚠️ ファイル中・旧コード中に `FreshKeep` という旧仮称が残っている箇所がありますが、**正式名称は「ころあい（Koroai）」** です。実装時はすべて「ころあい」に統一してください。

## About the Design Files
このバンドルに含まれる HTML/JSX ファイルは、**HTML で作成したデザインリファレンス（プロトタイプ）** です。見た目と挙動の意図を示すものであり、そのまま本番投入するコードではありません。

- プロトタイプは **ブラウザ内 Babel ＋ React 18 UMD** で動く単一ページのデモです（ビルド工程なし）。`window.FKT` / `window.FKD` というグローバル経由でテーマ・データを共有し、各 `fk-*.jsx` が `window` にコンポーネントを生やして連携する、デモ専用の構成になっています。
- **タスクは、これらのデザインをターゲットのコードベース（実際の本番環境）の確立されたパターン・ライブラリで作り直すこと** です。iOS ネイティブ（SwiftUI/UIKit）、React Native、Flutter など、プロジェクトの方針に合うフレームワークを選んで実装してください。環境がまだ無い場合は、最も適切なフレームワークを選定したうえで実装してください。
- `window` グローバル連携・ブラウザ内 Babel・`localStorage` 直書き等は **デモの都合** です。本番では各環境の標準的な状態管理・永続化・コンポーネント分割に置き換えてください。

## Fidelity
**ハイファイ（hi-fi）です。** 最終的な配色・タイポグラフィ・間隔・コンポーネント・主要インタラクションまで作り込まれています。UI はコードベースの既存ライブラリ／パターンを使って **ピクセル単位で再現** してください。配色・余白・角丸・影は本ドキュメントの「Design Tokens」を正とします。

## 画面構成（State Machine）

アプリのルートは画面ステート `screen` で切り替わります（`fk-app.jsx`）。
初回起動は `onboard`、2回目以降は `home`（`localStorage` の `onboarded` で判定）。

| screen | 内容 |
|---|---|
| `onboard` | ウェルカム（価値訴求の数ページ）→ 冷蔵庫セットアップ（今ある食材を好きなだけ選ぶ） |
| `home` | 今日の冷蔵庫（起点） |

ホームの上に重なるモーダル／オーバーレイ：

| モーダル | トリガー | 内容 |
|---|---|---|
| 追加 — 選ぶ | ホームの FAB（＋） | 種類別タイル。タップでカゴに1つ追加（詳細には飛ばない）。下部にカウントチップ＋「確認する」。**→ ADD_FLOW.md** |
| 追加 — 確認・編集 | 「確認する」 | カテゴリ別カードで名前・もち日数（ステッパー＋カレンダー）・残量（量/個数）を編集→冷蔵庫に追加。**→ ADD_FLOW.md** |
| 編集シート | ホームの食材カードをタップ | 名前・もち日数・残量を調整／削除 |
| 今朝のまとめ（ダイジェスト） | ホームの「今朝のまとめ」チップ | 食べ頃・期限間近をまとめて把握 |
| ふりかえり | ホームの食べきり記録カード | 累計・連続・マイルストーン |
| 設定 | ホーム右上の ⚙ | 表示・テーマ・月替わりリザルトの ON/OFF 等 |
| 月替わりリザルト | 月をまたいで初回起動（自動） | 先月の食べきり結果をリザルト形式で表示 |
| マイルストーン祝福 | 節目の食べきり数到達（自動） | 達成バッジ |
| 久しぶり起動 | 数日ぶりの起動（自動） | リセット／再開を提案 |

**全体像は `ころあい UIUXフロー図.html`（ストーリーボード）を参照。** 上段＝主要フロー、下段＝例外・空状態＋設計原則。

## 画面ごとの詳細

### ① 今日の冷蔵庫（ホーム）
- **目的**: 食べ頃が近い食材をひと目で把握し、何から使うか迷わせない。すべての動線のハブ。
- **レイアウト（上→下）**:
  - ヘッダー：左にキッカー（「こんにちは」等）＋タイトル「今日の冷蔵庫」、右に設定歯車（⚙、44px ヒットエリア）
  - メタ行：日付＋件数「6月10日（火）・冷蔵庫に N品」／右に「今朝のまとめ」チップ
  - 達成カード：今月の「食べきり」記録（大きな数字＋一言）。設定で表示 ON/OFF
  - セクション「食べ頃」：期限が近い食材カードのリスト
  - セクション「今週の食材」（旧称の揺れ注意。現行ラベルは tweak で可変、既定「今週の食材」）：ゆとりのある食材
  - FAB（＋）：画面下中央、テラコッタ円、48px 以上
- **食材カード（行）**: 左に円形カテゴリアイコン（単色文字グリフ、絵文字不使用）＋食材名＋右に残り日数バッジ。日数で色温度が変わる（後述 urgency）。
- **操作**: カードをタップ＝編集シート。スワイプ／ボタンで「食べた（食べきり）」「捨てた（処分）」。

### ② 食べものを追加 — カテゴリ選択
- **目的**: 買ってきた食材をすばやく登録。
- **レイアウト**: シートヘッダー（✕＋「食べものを追加」）／「かごに N品」バー／**種類ごとのセクション**（見出し＋3列タイルグリッド）。
  - セクション分け（確定仕様）: **肉・魚介**（魚・刺身／鶏肉／豚・牛肉）｜**野菜・きのこ**（葉物野菜／野菜／きのこ）｜**果物・乳製品**（果物／牛乳・乳製品）｜**大豆製品・卵**（豆腐・納豆／卵）｜**主食・惣菜**（パン／惣菜・弁当）。どのセクションにも入らないカテゴリは自動で「その他」。
- **タイル**: 角丸 20px、円形グリフアイコン、カテゴリ名、右上にカウントバッジ（未選択は「＋」、選択時はカテゴリ色で個数）。選択時はカテゴリ色の枠＋淡い塗り。
- **動線**: タイルをタップ → 詳細入力（最低限の情報を確認・追記して登録）。「かご」に複数ためて「かごに追加」でまとめてホームへ。

### ③ 追加 — 詳細入力 / 編集シート
- **目的**: もち日数（食べ頃）と残量を設定・調整。
- **コンポーネント**:
  - もち日数：行内ステッパー（−／「あと N日」／＋）。ボタンは 44px。
  - **カレンダー動線**: 「📅 カレンダーで日付を選ぶ」トグル → 月カレンダー。期限日を直接タップで設定。選択日はテラコッタ塗り、今日はテラコッタのリング、過去日はグレーアウト（選択不可）。**ステッパーと双方向同期**（日付↔日数を相互変換）。下部に「6/18（木）まで・あと8日」の確認表示。
  - **残量**: 2モードを切替（食材タイプで初期モードを出し分け）。
    - **ざっくり量**（amount）: 吸着スライダー。ストップは わずか／少し／半分／多め／満タン。
    - **個数**（count）: シンプルカウント（−／＋、分母なし）。単位はカテゴリ別（本・個・丁・枚 …）。編集時は初期量との差を色付き／空の丸（ピル）で表現。初期量を超えて足すと色付きの丸が増える。
  - 残量は **未設定OK（スキップ可）**。必須にしない。
- **CTA**: 「かごに追加」（テラコッタ塗りボタン）。

### ④ 今朝のまとめ（ダイジェスト）
- 食べ頃／期限間近の食材を、要点だけまとめて見せる軽い情報画面。ブランド名「ころあい」を表示。

### ⑤ ふりかえり ／ 月替わりリザルト
- **ふりかえり**: 累計食べきり数・連続記録・マイルストーン。深いグリーン地（`#41614a`→`#35513d`）に大きな数字＋食材色のドット＋一言（「先月より +5品。いいころあいです」）。
- **月替わりリザルト**: 月をまたいだ初回に自動表示。先月の結果をリザルト形式でポジティブに祝福し、今月も続けたくなる導線。設定でこのポップアップ自体を OFF にできる。

### 例外・空状態
- **空の冷蔵庫（初回）**: 「冷蔵庫はまだ空っぽ」＋「＋ 食べものを足す」。責めず最初の一歩へ誘導。
- **やさしいお知らせ（通知）**: 「そろそろ食べ頃 ◯◯／あと1日でおいしい食べ頃。今日のごはんに、どうですか？」急かさない言葉。
- **久しぶりに起動**: 「おかえりなさい」＋「そのまま続ける／リセットして始める」。

## Interactions & Behavior
- **追加フロー**: カテゴリタイル → 詳細 →「かごに追加」。複数ためてまとめて登録。完了時はトースト「N品を追加しました」でホームへ。
- **食べきり記録**: ホームで「食べた」→ 累計カウントを +1、トースト＋カウントアップ演出。称賛コピーはプールからランダム（直前と重複回避）。
- **カレンダー⇄ステッパー**: 双方向同期。日付選択→日数へ変換、日数変更→カレンダー選択日が移動。過去日は選択不可。
- **残量モード**: 食材タイプで初期モードを自動選択（例：牛乳・葉物→ざっくり量／バナナ・卵・豆腐→個数）。ユーザーはトグルで上書き可。選んだモードは記憶。
- **自動表示**: 月またぎ＝月替わりリザルト、節目到達＝マイルストーン、数日ぶり起動＝復帰提案。いずれも条件成立時のみ。
- **テーマ**: ライト／ダーク／ナイト（暖色を一段沈めた就寝前向け）。OS 設定追従も可。
- **アニメーション**: カウントアップ、トーストのフェード／スライド、シートのスプリング。装飾的な無限ループは避ける。

## State Management
プロトタイプは `fk-app.jsx` で集中管理（本番は各環境の標準手法へ）。主な状態：
- `screen`（`onboard` / `home`）
- `items`（冷蔵庫の食材配列。各要素は下記スキーマ）
- 食べきり累計・連続日数・最終起動日（リザルト／復帰判定用）
- 現在のモーダル（add / detail / edit / digest / review / settings / result / milestone / return）
- 設定（テーマ・パレット・各表示 ON/OFF・トーン）
- 永続化: プロトタイプは `localStorage`（キー `freshkeep_v1`）。本番は適切なストレージへ。

### 食材アイテムのスキーマ（`fkMakeItem`）
```
{
  id, catId, name,
  days,          // もち日数（残り日数）
  perishable,    // 生鮮フラグ（true=食べ頃セクション優先）
  addedAt,
  amtMode,       // 'amount' | 'count'
  amt,           // 0..1 の残量（amount モード）
  qty,           // 現在の個数（count モード）
  qtyTotal,      // 初期個数（編集時のピル表示用）
  unit,          // 本 / 個 / 丁 / 枚 ...
}
```

（別パレット "clay" も `fk-data.js` に定義。`FK_PALETTES` 参照。）

### 期限の色温度（urgency）— **赤は使わない**
残り日数を **セージグリーン → アンバー → テラコッタ** の暖色で表現。hue は `34 + (clamp(days,0,7)/7) * (150-34)`（0日=34 テラコッタ … 7日以上=150 グリーン）。OKLCH で算出（`fkUrgency`）：
- ライト pill 背景 `oklch(0.935 0.045 H)` / 文字 `oklch(0.46 0.105 H)` / solid `oklch(0.69 0.135 H)`
- ダーク pill 背景 `oklch(0.43 0.06 H)` / 文字 `oklch(0.90 0.085 H)` / solid `oklch(0.72 0.115 H)`

ソート用の tier: 0=今日 / 1=明日 / 2=〜3日 / 3=〜6日 / 4=7日以上。

### 残り日数ラベル（トーン別 `fkDayLabel`）
- gentle（既定）: 今日→「今日中に」／1日→「あすまで」／N→「あとN日」
- simple: 「今日」「N日」
- cheer: 「いま食べごろ」「そろそろ」「あとN日」「ゆっくりでOK」

### タイポグラフィ
- 和文/全体: **M PLUS Rounded 1c**（丸ゴシック）。weight 400/500/700/800。
- 欧文ワードマーク: **Quicksand**（500/600/700）。
- 本文 13–15px、見出し 16–24px、数字強調 22–64px。本番は各 OS の Dynamic Type / アクセシビリティ対応を推奨。

### 角丸・影・間隔
- 角丸: カード 12–14px、シート 18–24px、タイル 20px、ボタン 13–16px、円形アイコン 50%。
- アプリアイコンの角丸: かけない（iOS のスーパー楕円 ≈22.37% はシステム任せ）。
- 影: `0 6px 18px rgba(40,30,12,0.08)` 前後（カード）、シートはより強め。
- 間隔: 8 の倍数基調（6/7/8/10/12/14/18/22/26）。
- タップ領域: 最低 44px。
